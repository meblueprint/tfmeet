# 田径运动会管理系统 - 静态部署版

## 🎉 系统简介

这是一个完整的田径运动会管理系统，支持报名管理、赛程编排、成绩录入、奖状生成等核心功能。本版本为静态导出版本，可以直接在任何Web服务器上运行。

## 📦 部署方式

### 方式一：直接在本地浏览器运行（无需服务器）

1. 解压下载的压缩包
2. 双击打开 `index.html` 文件，即可在浏览器中运行

**注意**：由于使用 localStorage 存储数据，数据保存在您的浏览器中，不会上传到任何服务器。

### 方式二：部署到任意Web服务器

您可以将文件部署到以下任意环境：
- Apache HTTP Server
- Nginx
- IIS
- 任意静态网站托管服务（如 GitHub Pages、Vercel、Netlify 等）
- 轻量应用服务器

#### 部署到 Apache
```apache
# 将 out 目录下的所有文件上传到 Apache 的网站根目录
# 确保配置支持 SPA 路由
<Directory /var/www/html>
    RewriteEngine On
    RewriteBase /
    RewriteRule ^index\.html$ - [L]
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule . /index.html [L]
</Directory>
```

#### 部署到 Nginx
```nginx
# 将 out 目录下的所有文件上传到 Nginx 的网站根目录
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/out;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

#### 部署到轻量应用服务器（如腾讯云、阿里云）
1. 购买轻量应用服务器
2. 安装 Nginx 或 Apache
3. 将 out 目录下的所有文件上传到网站根目录
4. 配置服务器启动

### 方式三：本地启动简单HTTP服务器

如果您想在本地临时运行，可以使用以下方式：

#### 使用 Python
```bash
cd out
python -m http.server 8080
```
然后访问 http://localhost:8080

#### 使用 Node.js
```bash
cd out
npx serve -p 8080
```
然后访问 http://localhost:8080

#### 使用 PHP
```bash
cd out
php -S localhost:8080
```
然后访问 http://localhost:8080

## 📋 使用流程

### 第一步：登录系统
1. 打开系统，自动跳转到登录页
2. 使用 `admin/admin123` 登录（超级管理员）
3. 或点击"演示账户"按钮快速登录

### 第二步：配置运动会信息
1. 进入"运动会配置"页面
2. 填写运动会名称、届数、举办时间、学校名称等信息
3. 保存配置

### 第三步：管理班级和用户
1. 进入"班级管理"，添加参赛班级
2. 进入"用户管理"，添加班级管理员和学生账户

### 第四步：添加比赛项目
1. 进入"项目管理"
2. 添加比赛项目（如100米短跑、跳远等）
3. 设置计分规则（时间越小越好/距离越大越好等）

### 第五步：学生报名
1. 进入"报名管理"
2. 为学生添加报名记录
3. 审核报名信息

### 第六步：编排赛程
1. 进入"赛程管理"
2. 添加赛程，设置日期、时间、场地
3. 更新赛程状态（进行中/已完成）

### 第七步：录入成绩
1. 进入"成绩管理"
2. 为已完成的项目录入成绩
3. 系统自动计算排名和积分

### 第八步：生成奖状
1. 进入"奖状管理"
2. 批量生成获奖学生奖状
3. 预览、下载或打印奖状

### 第九步：查看统计
1. 进入"数据统计"
2. 查看班级总分排名、奖牌统计
3. 导出统计报表

## 💡 功能特点

✅ **纯本地运行**：无需数据库，所有数据存储在浏览器 localStorage
✅ **三种角色**：超级管理员、班级管理员、学生，权限分明
✅ **自动排名**：录入成绩后自动计算排名和积分
✅ **批量操作**：支持批量导入导出、批量生成奖状
✅ **数据导出**：支持导出 CSV 和 JSON 格式数据
✅ **操作日志**：所有操作都有记录，可追溯

## ⚠️ 注意事项

1. **数据存储**：本系统使用 localStorage 存储数据，数据保存在用户浏览器中
2. **数据备份**：建议定期使用"数据导出"功能备份数据
3. **浏览器兼容性**：推荐使用 Chrome、Edge、Firefox 等现代浏览器
4. **跨设备使用**：由于数据存储在本地，不同设备之间数据不共享
5. **存储容量**：localStorage 有容量限制（通常 5-10MB），大量数据请及时清理

## 🆘 常见问题

**Q: 数据会丢失吗？**
A: 数据保存在浏览器的 localStorage 中，不会丢失。但建议定期导出备份。

**Q: 如何清空所有数据？**
A: 在浏览器中按 F12 打开开发者工具，在控制台输入 `localStorage.clear()` 并回车。

**Q: 可以在手机上使用吗？**
A: 可以，但建议使用现代浏览器，如 Chrome、Safari 等。

**Q: 如何在其他电脑上使用？**
A: 需要部署到Web服务器上，然后通过浏览器访问。
